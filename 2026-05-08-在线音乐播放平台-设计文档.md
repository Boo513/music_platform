# 在线音乐播放平台 — 设计文档

## 1. 项目概述

构建一个在线音乐播放 Web 平台。用户上传 MP3 音乐，为每首歌标注音乐风格和情绪标签。平台核心差异化功能：播放音乐时，根据风格和情绪标签动态渲染沉浸式场景背景（赛博朋克霓虹城市、宇宙星河、山水森林等），配合粒子特效和动画，营造强烈的氛围沉浸感。

## 2. 技术选型

| 维度 | 选择 | 说明 |
|------|------|------|
| 平台形态 | Web 应用 | 响应式设计，PC 端为主 |
| 前端 | React + TypeScript | 生态完善，适合复杂交互 |
| 后端 | Java Spring Boot | REST API 服务 |
| 数据库 | MySQL | 关系型存储，5 张核心表 |
| 音乐来源 | 用户上传 | MP3 格式 |
| 风格/情绪识别 | 用户手动标签 | 上传时选择预设枚举值 |
| 用户系统 | 完整注册登录 | JWT 认证，多用户隔离 |

## 3. 架构设计

### 3.1 经典三层架构

```
React SPA (前端)
  ↕ REST API (JSON + multipart/form-data)
Spring Boot (后端)
  ↕ JDBC
MySQL (数据库)
  ↕ 文件系统
MP3 存储目录
```

- 前端负责 UI、播放器、氛围背景渲染（纯客户端 Canvas + CSS）
- 后端负责认证、文件管理、元数据 CRUD
- 前后端分离，独立开发测试

### 3.2 模块划分

**前端 4 模块：**
- 音乐播放器模块：MP3 解码播放、播放列表、进度控制、音量、播放模式
- 氛围背景引擎：根据 style+mood 标签，渲染对应场景背景 + 粒子特效
- 用户界面模块：登录注册、个人中心、播放历史、收藏管理
- 音乐管理模块：列表展示、搜索过滤、分类浏览、上传表单

**后端 4 服务：**
- 认证服务：注册/登录、JWT Token 管理、BCrypt 密码加密
- 文件服务：MP3 文件上传存储、音频流读取（支持 Range 请求）
- 音乐元数据服务：音乐信息 CRUD、风格/情绪标签、搜索 API
- 用户数据服务：播放历史、收藏、歌单 CRUD

## 4. 数据库设计

### 4.1 users 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 用户 ID |
| username | VARCHAR(50) UNIQUE | 用户名 |
| password_hash | VARCHAR(255) | BCrypt 加密 |
| avatar_url | VARCHAR(500) | 头像 URL |
| created_at | DATETIME | 注册时间 |

### 4.2 songs 表（中心表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 歌曲 ID |
| title | VARCHAR(200) | 歌曲标题 |
| artist | VARCHAR(100) | 歌手 |
| album | VARCHAR(100) | 专辑 |
| duration | INT | 时长(秒) |
| file_path | VARCHAR(500) | 文件路径 |
| cover_url | VARCHAR(500) | 封面图 |
| **style** | **VARCHAR(50)** | **音乐风格标签** |
| **mood** | **VARCHAR(50)** | **情绪标签** |
| uploader_id | BIGINT FK | 上传者 |
| created_at | DATETIME | 上传时间 |

### 4.3 playlists 表 + playlist_songs 关联表
歌单（id, name, user_id, is_public, created_at）↔ 歌曲，多对多关系。

### 4.4 play_history 表
记录每次播放（id, user_id, song_id, played_at）。

### 4.5 favorites 表
收藏关系（id, user_id, song_id, created_at）。

## 5. API 设计

所有接口前缀 `/api`，除 `/api/auth/*` 外均需 JWT Token（Header: `Authorization: Bearer <token>`）。

### 5.1 认证 `/api/auth`
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/register | 注册 |
| POST | /api/auth/login | 登录，返回 JWT |

### 5.2 歌曲 `/api/songs`
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/songs | 列表（?style=&mood=&keyword= 过滤） |
| GET | /api/songs/{id} | 详情 |
| POST | /api/songs | 上传（multipart/form-data: file + title + artist + album + style + mood） |
| PUT | /api/songs/{id} | 修改（仅上传者） |
| DELETE | /api/songs/{id} | 删除（仅上传者） |
| GET | /api/songs/{id}/stream | 音频流（支持 Range 断点续传） |

### 5.3 歌单 `/api/playlists`
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/playlists | 用户歌单列表 |
| POST | /api/playlists | 创建 |
| GET | /api/playlists/{id} | 详情（含歌曲） |
| POST | /api/playlists/{id}/songs | 添加歌曲 |
| DELETE | /api/playlists/{id}/songs/{songId} | 移除歌曲 |

### 5.4 用户数据 `/api/user`
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/user/profile | 个人信息 |
| GET | /api/user/history | 播放历史 |
| POST | /api/user/history | 记录播放 |
| GET | /api/user/favorites | 收藏列表 |
| POST | /api/user/favorites | 添加收藏 |
| DELETE | /api/user/favorites/{songId} | 取消收藏 |

## 6. 氛围背景引擎

### 6.1 标签枚举

**音乐风格（8 种）：** 摇滚 · 流行 · 古典 · 电子 · 民谣 · 爵士 · 嘻哈 · R&B

**情绪（6 种）：** 开心 · 悲伤 · 平静 · 激昂 · 浪漫 · 忧郁

### 6.2 场景映射（风格 → 背景场景）
| 风格 | 场景 |
|------|------|
| 摇滚 | 宇宙星河（星空、行星、星云） |
| 流行 | 粉色黄昏（暖色天空、浮云） |
| 古典 | 金色云层（暖金云海、光芒） |
| 电子 | 霓虹之城（赛博朋克城市、全息广告、飞车光轨） |
| 民谣 | 山水森林（绿意山谷、林间光斑） |
| 爵士 | 暮光之城（暗金城市天际线） |
| 嘻哈 | 夜间街道（暗色街景、霓虹灯牌） |
| R&B | 浪漫星海（粉紫星空、星尘漂浮） |

### 6.3 情绪对场景的影响
| 情绪 | 色彩变化 | 粒子表现 |
|------|----------|----------|
| 开心 | 明亮暖色，+20% 饱和度 | 快速上升光点，金色 |
| 悲伤 | 蓝灰滤镜，-30% 饱和度 | 缓慢下落，蓝色稀疏 |
| 平静 | 柔和色调，正常 | 缓慢呼吸式浮动，低密度 |
| 激昂 | 高对比度，+30% 亮度 | 快速爆发，火花四射 |
| 浪漫 | 粉紫暖色调 | 漂浮花瓣/心形，中速 |
| 忧郁 | 暗蓝灰，-40% 亮度 | 雨丝/下雪效果，慢速 |

### 6.4 渲染架构
- **背景层**：CSS 渐变 + DIV 构建的城市天际线/星空/山水等静态场景
- **粒子层**：CSS 动画驱动的浮动发光粒子，根据情绪调整速度/密度/颜色
- **雾效层**：模糊光晕叠加，营造景深和氛围
- 切歌时 1.5s cross-fade 过渡到新场景

### 6.5 性能策略
- 粒子数量控制在 9-15 个（CSS animation，非 Canvas 重绘）
- 使用 CSS transform + opacity 动画（GPU 加速）
- 场景切换时复用 DOM，仅更新颜色/位置参数

## 7. UI 设计

### 7.1 设计风格
- **主风格**：复古浪漫 + 赛博朋克融合
- **色调**：暖玫瑰粉、蜜桃橙、暮光紫为 UI 主色，场景背景独立配色
- **面板效果**：毛玻璃（backdrop-filter blur + 半透明背景 + 微光边框）
- **字体**：Clash Display（标题）、Satoshi（正文）、JetBrains Mono（数据）

- **90% 画面**：赛博朋克场景背景 + 粒子

- **半圆右侧**：上一首⏮ + 下一首⏭ + 随机🔀（统一 38px）
- **左上角**：大号音量图标 + 120px 滑条
- **右下角**：进度条贴底边，霓虹渐变色，发光拖拽点

### 7.3 首页布局
- 左侧 72px 图标式导航栏
- 主区域：顶部搜索 → 情绪快捷筛选（横向标签）→ 风格分类网格（4×2）→ 最近播放卡片列表（4列）
- 底部迷你播放条

### 7.4 登录/注册页
- 全屏场景背景 + 居中毛玻璃卡片
- 双面板切换登录/注册
- 品牌标识（音符图标）+ 发光按钮

### 7.5 上传页面
- 拖拽上传区（虚线边框 + 毛玻璃）
- 表单：歌曲名称、歌手、专辑、风格下拉、情绪下拉
- 底部提交按钮

### 7.6 个人中心
- 头像 + 用户信息 + 统计卡片（歌曲数/歌单数/播放数）
- Tab 切换：我的歌单 / 收藏 / 历史
- 列表式展示，毛玻璃卡片
