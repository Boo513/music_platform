## 2. 后端三层架构

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────────┐
│  请求流向：Controller → Service → Mapper → Database      │
│  响应流向：Controller ← Service ← Mapper ← Database      │
│                                                         │
│  ┌──────────────┐    ┌──────────────┐    ┌────────────┐ │
│  │  Controller  │───▶│   Service    │───▶│   Mapper   │ │
│  │  控制层       │◀───│   业务层      │◀───│   持久层    │ │
│  │              │    │              │    │            │ │
│  │ · 接收请求    │    │ · 业务逻辑    │    │ · SQL 映射  │ │
│  │ · 参数校验    │    │ · 事务管理    │    │ · 数据库交互 │ │
│  │ · 返回响应    │    │ · 数据组装    │    │ · CRUD 操作 │ │
│  └──────────────┘    └──────────────┘    └────────────┘ │
│                          │                              │
│                          ▼                              │
│                    ┌──────────────┐                     │
│                    │    Entity    │                     │
│                    │   实体层      │                     │
│                    │              │                     │
│                    │ · 数据库映射  │                     │
│                    │ · 纯数据对象  │                     │
│                    └──────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

### 2.2 各层职责与约束

| 层 | 职责 | 禁止 | 依赖方向 |
|---|------|------|----------|
| **Controller** | 接收HTTP请求、校验参数、调用Service、封装响应 | 不包含业务逻辑、不直接调用Mapper | → Service |
| **Service** | 业务逻辑编排、事务管理、数据校验、DTO转换 | 不处理HTTP请求/响应、不直接操作HttpServletRequest | → Mapper |
| **Mapper** | SQL映射、数据库CRUD、结果集映射为Entity | 不包含业务逻辑、不调用其他Mapper | → Entity |
| **Entity** | 纯数据对象，与数据库表字段一一映射 | 不包含业务方法（仅getter/setter） | 无 |

### 2.3 调用链路示例：歌曲播放

```
用户请求: GET /api/songs/1/stream
              │
    ┌─────────▼─────────┐
    │ SongController    │  ← 控制层：接收请求，从SecurityContext获取userId
    │ .stream(id,       │
    │  request,         │
    │  Authentication)  │
    └─────────┬─────────┘
              │ 调用
    ┌─────────▼─────────┐
    │ SongService       │  ← 业务层：校验歌曲是否存在、构建ResourceRegion、
    │ .getStreamData    │    处理Range请求头、返回ResourceRegion
    │ (id, httpHeaders) │
    └─────────┬─────────┘
              │ 调用
    ┌─────────▼─────────┐
    │ SongMapper        │  ← 持久层：SELECT * FROM songs WHERE id = ?
    │ .selectById(id)   │    返回Song实体
    └─────────┬─────────┘
              │ 返回
    ┌─────────▼─────────┐
    │ Song Entity       │  ← 实体层：id, title, file_path, duration...
    └───────────────────┘
```

### 2.4 完整调用示例：上传歌曲

```
POST /api/songs/upload (multipart/form-data)
              │
    ┌─────────▼──────────────────────────────────────┐
    │ SongController.upload()                        │
    │  1. @Valid 校验 title/artist/style/mood 非空    │
    │  2. 从 Authentication 获取 userId               │
    │  3. 调用 songService.upload(...)                │
    │  4. 将返回的 Song 转为 ApiResponse 返回          │
    └─────────┬──────────────────────────────────────┘
              │
    ┌─────────▼──────────────────────────────────────┐
    │ SongService.upload(title, artist, style,        │
    │                    mood, file, userId)          │
    │  1. 生成 UUID 文件名 → uploads/audio/uuid.mp3    │
    │  2. Files.copy() 写入磁盘                       │
    │  3. 获取文件时长（JAudioTagger解析MP3）           │
    │  4. 构建 Song 实体，调用 songMapper.insert(song) │
    │  5. 返回完整的 Song 对象                         │
    │  @Transactional ← 文件写入+数据库插入原子操作     │
    └─────────┬──────────────────────────────────────┘
              │
    ┌─────────▼──────────────────────────────────────┐
    │ SongMapper.insert(Song song)                   │
    │  INSERT INTO songs (title, artist, file_path,   │
    │    duration, style, mood, uploader_id)          │
    │  VALUES (#{title}, #{artist}, ...)              │
    └────────────────────────────────────────────────┘
```

### 2.5 包结构对应

```
com.musicplatform/
├── controller/    ← 控制层：@RestController, @RequestMapping
├── service/       ← 业务层：@Service, @Transactional
├── mapper/        ← 持久层：@Mapper (MyBatis接口)
├── entity/        ← 实体层：纯POJO，对应数据库表
├── dto/           ← 数据传输对象：Request/Response DTO
├── config/        ← 配置类：Security, CORS, MyBatis
├── security/      ← 安全组件：JWT, Filter
└── exception/     ← 异常处理：全局异常处理器
```

## 4. 数据库设计

### 4.1 DDL

```sql
CREATE DATABASE music_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE music_platform;

-- 用户表
CREATE TABLE users (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50)  NOT NULL UNIQUE,
    password    VARCHAR(255) NOT NULL COMMENT 'BCrypt加密',
    nickname    VARCHAR(100),
    avatar      VARCHAR(500) COMMENT '头像URL',
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB;

-- 歌曲表
CREATE TABLE songs (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    title       VARCHAR(200) NOT NULL,
    artist      VARCHAR(200) NOT NULL DEFAULT '未知艺术家',
    file_path   VARCHAR(500) NOT NULL COMMENT 'MP3文件路径（相对于uploads/audio/）',
    cover_path  VARCHAR(500) COMMENT '封面图路径',
    duration    INT NOT NULL DEFAULT 0 COMMENT '时长（秒）',
    style       VARCHAR(20) NOT NULL COMMENT '风格: rock/pop/classical/electronic/folk/jazz/hiphop/rnb',
    mood        VARCHAR(20) NOT NULL COMMENT '情绪: happy/sad/calm/excited/romantic/melancholy',
    uploader_id BIGINT NOT NULL,
    play_count  INT NOT NULL DEFAULT 0,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (uploader_id) REFERENCES users(id),
    INDEX idx_style (style),
    INDEX idx_mood (mood),
    INDEX idx_style_mood (style, mood),
    INDEX idx_play_count (play_count DESC)
) ENGINE=InnoDB;

-- 歌单表
CREATE TABLE playlists (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(200) NOT NULL,
    description TEXT,
    cover       VARCHAR(500),
    user_id     BIGINT NOT NULL,
    is_public   TINYINT(1) NOT NULL DEFAULT 1,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB;

-- 歌单-歌曲关联表
CREATE TABLE playlist_songs (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    playlist_id BIGINT NOT NULL,
    song_id     BIGINT NOT NULL,
    sort_order  INT NOT NULL DEFAULT 0,
    added_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (playlist_id) REFERENCES playlists(id) ON DELETE CASCADE,
    FOREIGN KEY (song_id) REFERENCES songs(id) ON DELETE CASCADE,
    UNIQUE KEY uk_playlist_song (playlist_id, song_id)
) ENGINE=InnoDB;

-- 收藏表
CREATE TABLE favorites (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id     BIGINT NOT NULL,
    song_id     BIGINT NOT NULL,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (song_id) REFERENCES songs(id),
    UNIQUE KEY uk_user_song (user_id, song_id)
) ENGINE=InnoDB;

-- 播放历史表
CREATE TABLE play_history (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id     BIGINT NOT NULL,
    song_id     BIGINT NOT NULL,
    played_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (song_id) REFERENCES songs(id),
    INDEX idx_user_played (user_id, played_at DESC)
) ENGINE=InnoDB;
```

## 5. 后端 API 设计

### 5.1 统一响应格式

**通用响应体：**

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**分页响应体：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "size": 20,
    "pages": 5
  }
}
```

**错误响应体：**

```json
{
  "code": 401,
  "message": "Token已过期或无效",
  "data": null
}
```

### 5.2 HTTP 状态码规范

| 场景 | code |
|------|------|
| 请求成功 | 200 |
| 创建资源成功 | 200（不使用 201） |
| 参数校验失败 | 400 |
| 未认证（无 Token / Token 无效） | 401 |
| 无权限（非资源所有者） | 403 |
| 资源不存在 | 404 |
| 资源冲突（重复收藏/重复用户名） | 409 |
| 服务器内部错误 | 500 |

### 5.3 全局错误码

| 错误码 | 说明 |
|--------|------|
| `USERNAME_EXISTS` | 用户名已被注册 |
| `INVALID_CREDENTIALS` | 用户名或密码错误 |
| `TOKEN_EXPIRED` | Token 已过期 |
| `TOKEN_INVALID` | Token 格式错误 |
| `SONG_NOT_FOUND` | 歌曲不存在 |
| `FILE_NOT_FOUND` | 音频文件丢失 |
| `INVALID_STYLE` | 风格枚举值非法 |
| `INVALID_MOOD` | 情绪枚举值非法 |
| `FILE_TOO_LARGE` | 文件超过大小限制 |
| `UNSUPPORTED_FORMAT` | 仅支持 MP3 格式 |
| `FAVORITE_EXISTS` | 已收藏该歌曲 |
| `FAVORITE_NOT_FOUND` | 未收藏该歌曲 |

**成功响应 — 有 Range (206)：**

```
Accept-Ranges: bytes

```

**成功响应 — 无 Range (200)：**
```
HTTP/1.1 200 OK
Content-Type: audio/mpeg
Content-Length: 5242880
Accept-Ranges: bytes

[全部 MP3 二进制数据]
```

**后端实现要点：**
```java
// 核心逻辑
public ResponseEntity<ResourceRegion> stream(Long id, String rangeHeader) {
    Song song = songMapper.selectById(id);
    Path filePath = Paths.get(audioDir, song.getFilePath());
    Resource resource = new FileSystemResource(filePath);
    long fileSize = resource.contentLength();

    if (rangeHeader == null) {
        return ResponseEntity.ok()
            .header(HttpHeaders.CONTENT_TYPE, "audio/mpeg")
            .header(HttpHeaders.ACCEPT_RANGES, "bytes")
            .body(resource);
    }

    // 解析 Range: bytes=start-end
    RangeResourceRegionConverter converter = new RangeResourceRegionConverter();
    ResourceRegion region = converter.toRegion(resource, rangeHeader);

    return ResponseEntity.status(HttpStatus.PARTIAL_CONTENT)
        .header(HttpHeaders.CONTENT_TYPE, "audio/mpeg")
        .contentLength(region.getCount())
        .body(region);
}
```

**错误响应：**
```json
// 404 — 歌曲不存在
{ "code": 404, "message": "歌曲不存在", "data": null }

// 404 — 文件丢失
{ "code": 404, "message": "音频文件不存在", "data": null }
```

**注意：** 流式接口返回的是 MP3 二进制数据，响应体格式不是 JSON，错误时返回 JSON。前端 `<audio>` 标签遇到 400/404 时需监听 `error` 事件做降级处理。

### 6.3 JWT 核心实现要点

```java
// JwtTokenProvider.java — 关键方法
public String generateToken(Long userId, String username) {
    return Jwts.builder()
        .subject(String.valueOf(userId))
        .claim("username", username)
        .issuedAt(new Date())
        .expiration(new Date(System.currentTimeMillis() + expirationMs))
        .signWith(secretKey)
        .compact();
}

public Long getUserIdFromToken(String token) {
    return Long.parseLong(
        Jwts.parser().verifyWith(secretKey).build()
            .parseSignedClaims(token).getPayload().getSubject()
    );
}
```

## 10. 核心交互流程

### 10.1 播放一首歌的完整数据流

```
用户点击歌曲卡片
    ↓
playerStore.play(song, queue)
    ↓
更新 currentSong, queue, isPlaying: true
    ↓
PlayPage 检测 currentSong 变化
    ↓
SceneEngine 根据 currentSong.style + currentSong.mood 切换场景
    ↓
<audio> src 指向 GET /api/songs/{id}/stream
    ↓
浏览器自动发送 Range 请求 → Spring Boot 返回 206 Partial Content
    ↓
音频开始播放，场景粒子动画启动
    ↓
每 250ms 更新 playerStore.currentTime
    ↓
POST /api/history/{songId} 记录播放
```

### 10.2 上传一首歌的流程

```
UploadPage
    ↓
用户拖入 MP3 文件 + 选择 style/mood + 填写 title/artist
    ↓
POST /api/songs/upload (multipart/form-data)
    ↓
后端:
  1. 接收 MultipartFile
  2. 保存到 uploads/audio/{uuid}.mp3
  3. 用 JAudioTagger 或 FFmpeg 获取时长
  4. 写入 songs 表
  5. 返回 Song 对象
    ↓
前端跳转到首页，新歌曲出现在列表中
```

## 12. 注意事项

1. **Range 请求**：Spring Boot 需要手动实现 `ResourceRegion` 返回，默认的 `ResourceHttpMessageConverter` 不自动支持 Range
2. **文件路径安全**：MP3 文件存储于 `uploads/` 目录，需在 WebMvcConfig 中配置资源映射，防止路径遍历攻击
3. **场景内存管理**：react-three-fiber Canvas 卸载时需 dispose geometries/materials，避免 WebGL 上下文泄漏
4. **<audio> 自动播放**：浏览器限制自动播放，用户首次需手动点击播放，后续可通过 JS 切换 src
5. **SimplexNoise**：开发文档中已有独立实现，迁移到 TypeScript 时保持算法不变
6. **CORS**：开发时 Vite dev server 在 5173，Spring Boot 在 8080，需配置 CORS 或 Vite 代理
